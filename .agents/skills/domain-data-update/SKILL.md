# 领域数据维护规范（模板）

> 用于替代任何仓库绑定的数据技能（如特定知识树、特定配置树）。

## 适用范围

- 核心领域数据文件
- 数据同步脚本输入输出
- 依赖该数据的读取模型

## 基本规则

1. 数据主键命名规则必须固定且可验证。
2. 变更必须先通过结构校验脚本。
3. 若涉及 ID 变更，必须全局引用检查。
4. 删除操作需可回滚（软删除/备份）。

## 最低验证

```bash
bash scripts/check_errors.sh
npm test
```

## 交付要求

- 更新数据变更说明
- 记录影响范围和回滚策略
